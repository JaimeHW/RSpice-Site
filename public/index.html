03 / ANALYSIS MAP

Implemented analyses and current exposure.

Core analyses run through the native engine. Selected statistical, periodic, and RF paths use deck, CLI, or IDE-specific entry points; the status column distinguishes complete paths from partial kernels.

FAMILYCOMMAND / ROUTEENGINE BEHAVIORSTATUS

Operating point & sweeps

.OP .DC .STEP .TEMP .TF .SENS

IMPLEMENTED DEVICE, DECK, AND HOST FEATURES

DEVICE → DECK → SOLVE → EXPORT
01 / SEMICONDUCTOR MODELS

MOS and bipolar models

Native models cover the principal silicon paths; bundled Verilog-A sources provide generated implementations for additional compact-model families.

  • BSIM4 v4.8, BSIM3v3.3, VDMOS, EKV3, MOS1/2/3/6, BSIM1/2
  • Generated BSIM-SOI and EKV 2.6 built-ins where sources are bundled
  • Gummel-Poon plus generated VBIC, HICUM, and MEXTRAM where sources are bundled
02 / DEVICES & INTERCONNECT

Junction, magnetic, and line devices

Device registration includes junction models, magnetic components, and ideal or lossy transmission-line elements.

  • Diode, JFET/JFET2, MES/MESA/HFET families, and GaN HEMT
  • Temperature-aware R/C/L, coupled and multi-winding transformers
  • Jiles–Atherton saturable inductors; ideal, LTRA, TXL, and CPL lines
03 / SOURCES & MIXED SIGNAL

Sources, controlled elements, and bridges

Independent and controlled sources provide time-domain stimulus, behavioral expressions, switching, and analog/digital conversion.

  • PULSE, SIN, EXP, PWL, SFFM, AM, TRNOISE, and PWL file sources
  • E/F/G/H controlled and B behavioral sources; controlled switches
  • XSPICE-style analog/digital elements, tri-state drivers, and A/D–D/A bridges
04 / DECK & MODEL PIPELINE

Netlist elaboration and Verilog-A

The parser resolves parameters, includes, libraries, and subcircuits before matrix assembly; Verilog-A uses interpreted, JIT, and generated-Rust paths.

  • .SUBCKT, .PARAM, expressions, .INCLUDE, and .LIB
  • .OPTIONS, .TEMP, engineering suffixes, and flattened elaboration
  • Verilog-A parser, semantic IR, bytecode VM, native JIT, and generated-Rust backends
05 / AUTOMATION & VALIDATION

CLI reports and regression tests

CLI exit codes and report formats allow measurement failures, timeouts, invalid results, and golden-reference mismatches to stop CI jobs.

  • run, info, check, compare, convert, and compile-va
  • Typed exit codes, timeouts, JUnit, TAP, JSON summaries, and measurement files
  • ngspice and Xyce corpus harnesses with explicit oracles and failure tracking
06 / INTERFACES & DATA

Desktop, Python, WASM, and formats

The egui application, CLI, Python bindings, and WebAssembly bindings use the same core, with target-specific analysis exposure.

  • egui/wgpu desktop Workbench, CLI, Python/NumPy, and WebAssembly bindings
  • SPICE RAW, ASCII RAW, CSV, TSV, JSON, HDF5, and Touchstone output
  • Native desktop is the primary target; browser support is experimental and mobile UI remains a design target

05 / HOSTED SERVICES

Local projects remain usable without hosted services.

Planned services add identity, project synchronization, collaboration, managed simulation, and public or private result links. Local files and local simulation remain independent of those services.

04 / PLANNED

Managed simulation

Queued runs, runner selection, cost estimates, retained outputs, and shareable result URLs.

ACCOUNT LOCATION

Accounts belong to the application service.

The public site remains account-free. Registration, organization management, billing, and authenticated project access will use the hosted application.

SHARING POLICY

Every shared URL includes visibility controls.

Planned result pages distinguish public, unlisted, and private access and retain the input revision, run configuration, expiration, revocation, and export settings.

COMPUTE BILLING

Runner time and storage are separate usage.

Local simulation does not require a hosted plan. Managed corner, Monte Carlo, and optimization jobs show estimated runner and storage cost before submission.
